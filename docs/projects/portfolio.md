# Créer votre portfolio web personnel (300 points)

## Contexte

Dans le domaine du design et du développement web, votre portfolio est votre carte de visite numérique. C'est souvent la première impression que vous donnez à un recruteur ou un client potentiel. Un portfolio efficace ne se contente pas de montrer ce que vous avez fait, il démontre qui vous êtes en tant que professionnel et comment vous pensez.

## Objectif du projet

Créer une page web portfolio qui présente vos compétences, votre expérience et vos projets de manière professionnelle et attrayante. Cette page servira de vitrine pour votre travail et facilitera votre recherche de stage, d'emploi ou de projets freelance.

## Livrables attendus

1. **Une page web fonctionnelle** (HTML/CSS minimum, JavaScript optionnel)
3. **Hébergement en ligne** (GitHub Pages, Netlify, Vercel, ou autre)

## Sections obligatoires

Votre portfolio doit inclure au minimum les sections suivantes :

### 1. Page d'accueil / À propos
- Votre nom et titre professionnel
- Photo professionnelle ou avatar
- Phrase d'accroche décrivant votre profil en 1-2 lignes
- Brève biographie (150-300 mots)
- Vos domaines d'expertise

### 2. Compétences
- Compétences techniques (langages, outils, logiciels)
- Compétences transversales (soft skills)
- Niveau de maîtrise pour chaque compétence (optionnel)

### 3. Projets / Réalisations
- Minimum 1 projet présenté
- Pour chaque projet :
  - Titre et description
  - Votre rôle
  - Technologies utilisées
  - Visuel (capture d'écran, mockup)
  - Lien vers le projet ou GitHub (si applicable)

### 4. Contact
- Formulaire de contact OU adresse email cliquable
- Liens vers vos réseaux professionnels (LinkedIn, GitHub, Behance, etc.)
 
## Ressources et outils recommandés
### Inspiration
- **Awwwards** - portfolios primés
- **Dribbble** - designs de portfolios
- **Behance** - projets de designers
- **One Page Love** - portfolios one-page

### Frameworks et outils
- **HTML/CSS pur** - pour apprendre les bases
- **Bootstrap ou Tailwind CSS** - pour accélérer le développement
- **Figma** - pour maquetter avant de coder
- **GitHub Pages** - hébergement gratuit
- **Netlify ou Vercel** - déploiement facile

## Checklist : Les éléments d'un portfolio réussi

### CONTENU

#### Informations essentielles
- [ ] Nom et titre professionnel clairement visibles
- [ ] Biographie concise et engageante (qui êtes-vous, ce qui vous passionne)
- [ ] Proposition de valeur claire (ce que vous apportez)
- [ ] Compétences techniques listées et organisées
- [ ] Au moins 3 projets de qualité présentés
- [ ] Coordonnées facilement accessibles

#### Qualité des projets présentés
- [ ] Chaque projet a un titre descriptif
- [ ] Contexte et objectifs du projet expliqués
- [ ] Votre rôle et contributions spécifiques clarifiés
- [ ] Technologies et outils utilisés mentionnés
- [ ] Résultats ou impacts du projet (si applicable)
- [ ] Visuels de haute qualité (screenshots, mockups, photos)
- [ ] Liens vers projets en ligne ou code source (GitHub)

#### Texte et communication
- [ ] Ton professionnel mais authentique
- [ ] Textes sans fautes d'orthographe ou de grammaire
- [ ] Phrases courtes et claires
- [ ] Jargon technique expliqué si nécessaire
- [ ] Appel à l'action (CTA) sur la page de contact

### DESIGN ET FORME

#### Identité visuelle
- [ ] Palette de couleurs cohérente (2-4 couleurs maximum)
- [ ] Typographie lisible et professionnelle (2-3 polices maximum)
- [ ] Style graphique cohérent sur toute la page
- [ ] Identité visuelle qui reflète votre personnalité professionnelle
- [ ] Logo ou signature personnelle (optionnel mais recommandé)

#### Layout et structure
- [ ] Hiérarchie visuelle claire (titres > sous-titres > texte)
- [ ] Navigation intuitive et accessible
- [ ] Espacement généreux entre les sections
- [ ] Grille ou système d'alignement cohérent
- [ ] Pas de surcharge visuelle (espaces blancs bien utilisés)
- [ ] Longueur de ligne de texte confortable (50-75 caractères)

#### Images et médias
- [ ] Photos et visuels de haute qualité
- [ ] Images optimisées pour le web (poids réduit)
- [ ] Alt text pour toutes les images (accessibilité)
- [ ] Visuels pertinents et professionnels
- [ ] Ratio d'aspect cohérent pour les images de projets

#### Interactivité et animations
- [ ] Liens clairement identifiables (couleur, soulignement, ou hover)
- [ ] Effets de survol (hover) sur les éléments cliquables
- [ ] Animations subtiles et non distrayantes
- [ ] Transitions fluides entre les sections
- [ ] Pas d'animations automatiques excessives
- [ ] Site hébergé en ligne et accessible publiquement
